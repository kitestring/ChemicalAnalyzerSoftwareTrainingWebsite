Utilizing the Acquisition Queue

Introduction

The following section explains how to complete the acquisition queue in order to schedule an acquisition. The acquisition queue can save settings that will be used for repeated acquisitions. This section assumes the user already has an understanding of how to build methods.

A live acquisition monitored using a chromatogram and user specified extracted ions.

Columns

The table below provides a brief description for each of the columns found in the acquisition queue.

Column Description
AS Method Select the Autosampler Method for the acquisition if using an autosampler; leave the cell empty if an autosampler will not be used.
DP Method Select a Data Processing Method if a Data Processing Method is being used. When a DP Method is entered into the acquisition queue, the data will be processed automatically after acquisition. Leave the cell empty if a Data Processing Method is not being used.
Folder The name of the folder where the sample file will be stored should display in the Folder column. If the name of the sample folder does not display, select the sample folder from the list.
GC Method Select the desired Chromatographic Method if using a chromatograph; leave the cell empty if a chromatograph will not be used.
MS Method Select the desired Mass Spectrometer Method. An MS Method is the only method required for analysis.
Name Select in the cell and enter the desired name for the sample file that will be created.
QC Method Select the QC method if one will be used for the sample. It is recommended to create a blank QC Method to use for acquisition when QC tunes will not be automatically scheduled.
Repetitions Enter the number of times the sample should be acquired.
Resume Time If a delay is being utilized to schedule an acquisition. This column lists the data and time the scheduled start will begin.
Status As a read only cell, it displays the instrument's condition.
Tray Enter the tray where the sample is located.
Type Select the type of analysis from the list. The list below this table provides a description of each of the analysis types.
Vial
(Autosampler)		 If using an autosampler, enter the number for the vial location in the Autosampler Tray.
Vial
(L-PAL3)		 For the L-PAL3, the current Vial cell will show dots at the right side of the Vial cell. Select the dots to display a dialog showing a graphic of the current tray configuration. The desired vial can be selected from this graphic.

Below is a list of each analysis types which is defined using the type column in the acquisition queue.

  • Sample: Analyzes a sample.
  • Auto-Tune: Automatically runs the recommended set of tunes for Pegasus BT.
  • Source and Analyzer Focus: Adjusts the flight tube voltages in order to optimize the intensity, resolution, and symmetry of the spectral peaks produced by the instrument.
  • Mass Calibration: Derives an equation for converting between TOF and m/z.
  • Gain Optimization: Adjusts the detector voltage to optimize its response to ion strikes.
  • Detector Measurement: Measures the detector voltage without adjusting it.
  • Leak Check: Examines the data coming off the instrument for indications that there is a leak somewhere in the system.
  • Tune Validation: Provides qualitative information regarding the current tune quality.
  • Pause: Stops acquisition when the acquisition queue reaches that row until restarted by the user.
  • Delay: Holds the queue until a specified time.
  • Defrost: Used to melt the ice and evaporate the water that may have condensed in the cold jet tubing within the dewar/heat exchanger before a complete plug forms.
