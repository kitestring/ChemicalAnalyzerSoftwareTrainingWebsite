Tune Validation

Introduction

To ensure the quality of the acquired data perform a Tune Validation (Daily Instrument Check) daily. A tune validation is typically acquired as the last step of a full system tune, and the data generated is used to generate a performance report. Performance reports are designed to provide qualitative information regarding the current state of the instrument. Currently, there is one type of performance report, the Tune Validation Performance Report.

Performance reports are based on the immediate data collection capabilities of the data acquisition system. Post-collection data processing, such as mass calibration, should not be performed on the raw samples collected as this will produce ambiguity between what the user sees while viewing the performance report vs. viewing the sample directly. Information obtained during the collection is set aside and stored separately, and as such, the collected sample can be freely removed from the database afterwards.

Understanding the Tune Validation Performance Report

The Tune Validation Performance Report gives a quick snapshot of the instrument’s current performance in regards to tune quality. A 90 second raw acquisition is initiated and spectral analysis is performed upon the data. In the following sections, each area of the report will be explained in more detail. The results on the Tune Validation Report are scaled to 10 spectra per second collected at 1 KHz to allow meaningful comparison of reports whose data was collected using different settings.

Tune Validation Performance reports are based on the expectation that PFTBA is used as the tune compound within the system. As such, during data analysis, mass 69 is used as the reference mass. The criteria assume the tune mass in the MS method is mass 219; should another tune mass be used, the criteria may incorrectly report pass/fail because they are based on 219.

  • Information
    • The information section of the Tune Validation Performance Report provides the basic information regarding when the report was generated and which methods were used in the collection. Status information regarding the last time the instrument was tuned with actions such as Leak Check, Mass Calibration, Gain Optimization, and Source and Analyzer Focus are displayed within the information section. This area also displays the pass/fail status of the test.
  • Instrument Parameters
    • The instrument parameters table shows the hardware conditions that existed when the report was generated. It includes various temperatures, flows, pressures, voltages, filament, and other information.
  • Summary
    • The summary is comprised of four components:
      1. Recommended Action
      2. Table of Tune Mass Results
        • Mass Accuracy is for the tune mass, while Mass Accuracy RMS is the root mean square of all the masses in the mass calibration table.
      3. Leak Check Results & Pass Fail Criteria Table
        • Note, the default pass fail criteria can be edited using the Quality Control method. For more information refer to the Leak Check tuning section.
      4. Spectra plot of the focus tune mass
        • This allows a quick, qualitative look at the peak shape to check for tailing.
  • Details
    • The observed masses, and mass accuracies of all masses present in the mass calibration table will be displayed in the Details section.
    • Information is presented not only for the selected report object, but also for the most recent report. This addition gives the added dimension of how time is affecting the instrument’s performance.
    • The report contains the isotopic pattern table. This table is also based on the mass calibration table entries described previously. Should those entries contain formula information, the M and M+1 masses will be searched for in the raw data collection. The information present in this table is as follows:
      • Formula
      • M and M+1 values
      • Observed isotopic mass for each M and M+1
      • Intensity for both isotopes
      • Ratio of intensities from the M and M+1 isotopes
      • Expected ratio of those intensities
      • Numeric difference and percent error of observed ratio to expected ratio
    • The details section also contains a table describing the resolution, intensity, and intensity ratio (with respect to m/z 69) of all masses from the mass calibration table. This table, like the isotopic pattern table, will display information from the most recent report.
  • Spectrum
    • The Spectrum section presents a mass-range limited spectra plot based on the actual acquired data. This gives the report additional qualitative feedback supporting the information presented in previous report sections.
